# vuepress踩坑记

# 引入自定义主题

打开vuepress官方文档关于主题的页面.
0.0 发现里面并没有教我们怎么修改默认的主题.

其他第三方的主题倒是能够在在各大大佬那边搞到，看了半天也不知道怎么样能够修改。（这时候还没有看到下面的继承主题的内容可能是眼瞎了）使用这些大佬写的漂亮的主题也不是什么太大的难事，但这并不是我们想要的。

习大大告诉我们：我们仍要继续自力更生，核心技术靠化缘是要不来的！ 我想要的是自己动手修改，通过修改来学习怎样开发一个主题， 再看官方文档，只有个开发主题

进去看看，让我在文档根目录创建一个.vuepress/theme目录，接着创建一个 Layout.vue文件：

.
└─ .vuepress
   └─ theme
       └─ Layout.vue

听起来好像很简单
然后我就照做了，开启项目，打开页面....发现页面一片空白。突然感觉自己那么没有文化被骗了。

............................................................

后来经过我多方化缘终于领悟到了一些门道：
原来主题这东西是包括了各个页面的组件，当我在.vuepress/theme目录中加了内容之后，则会覆盖掉默认的主题，所以我这样子是渲染不出来的。

默认主题就存放在node_modules里面

.项目根目录
└─ .node_modules
   └─ @vuepress
       └─ theme-default

把这个theme-default文件夹里的内容拷到刚才说的/.vuepress/theme文件夹下，就可以相当于在项目中引入了一个自定义主题。开启项目：

TIP

当出现 tip Apply local theme at ....vuepress\theme... 就表明已经成功的加载了theme文件夹中的主题了

# 主题修改

终于到了我最熟悉的修改环节，有了代码可以参考，搞起来就简单很多了，虽然这些主题是vue写的，但凭着我写过一些react项目积累的经验，还是没啥难度的，摸着石头过河踩踩坑基本就能过去了。

敢于试验，不能像小脚女人一样。看准了的，就大胆地试 ——邓小平

用了张自己的图片，结果发现样式不好(如下图)，怎么这样小家子气，宽度要拉满

找了找，靠着我那可怜的英语，依稀能够辨认出是写在.vuepress/theme/components/Home.vue组件里的.
页面下面写了style，这个style好像写法有点独特，style标签上好像是这种语法的名字 <style lang="stylus"> 估计又是哪家的语法糖，在下面照着样子加一下
然后踩坑了，页面又变一片空白了，控制台报错了，应该是有些语法的问题。

还是老老实实先稍微看下stylus文档，然后在他的基础上改吧，

.home
  padding $navbarHeight 0 0 0
  width 100%
  display block
  .hero
    img
      /*  max-width  100%; */
      /* max-height  280px; */
      display: block;
      /* margin: 3rem auto 1.5rem; */

banner宽度拉满，高度自适应，要搞得有点扁平化，不要的样式都改掉。然后发现这里的修改居然是热更新的（主要是之前玩的地方的很多都不是热更新的，所以有些惊讶）

TIP

这里顺道说一下，这个项目其实是一个vue项目。所以vue组件能热更新是在正常不过的事情了，那些markdown文件好像也是可以热更新的，只有部分地方不能热更新。比如侧边栏里面之类的，这些是在项目启动或者打包的时候加进去的所以修改md文件里的h1 h2标签时，侧边栏是不会更新的，我好像发现只能重启项目才会更新。还有一些config.js文件的修改同样也是不能更新的。

回归刚才

在.vuepress/styles/palette.styl中可以配置accentColor textColor borderColor codeBgColor 但其实这四个少的可怜，大部分地方其实都不能修改，去源码里修改样式其实是很容易的。

不过既然是要修改默认主题，那就不能简单的只是修改下他的源代码了，这样其实和vuepress前辈大佬们的思想不太符合，我们修改是需要能有更多的属性可以让用户配置，而不是让用户来改源代码。

带着这样的目标我们继续看代码，在某些样式代码中我们看到了这样的代码：

color $accentColor

然后在.vuepress/styles/palette.styl 中我们能看到：

$accentColor = #e60020

这样类似的文件中可以编辑设置css的值，这个值将会用到项目中每一个地方，就好像css中也引入了全局的css值变量 依样画葫芦，应该就可以了，然后我发现了一些别的地方
然后，我发现默认主题下有一个description属性，写在项目的.vuepress/config/config.js中，在.vuepress/theme/components/Home.vue中能找到这样的代码

<h3 class="nav-description">
  {{ data.tagline || $description || 'Welcome to your VuePress site' }}
</h3>

这里你可以看到有data.属性和$...是两种方法，然后找一下发现其实data这玩意的属性是在全局的REAMDE.md 里面写的，这样我们就可以获得用户定义的内容，现在可以完成包括修改样式自定义渲染修改内容这些操作

# 继承主题

在踩过这许多坑之后，我终于找到了主题继承的方法 。 可以参考官网继承主题部分
在 theme 新建 index.js

module.exports = {
  extend: '@vuepress/theme-default'
}

比如说这样其实就能继承官方默认的主题
